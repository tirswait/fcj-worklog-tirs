In this section, we will create an **IAM User**, assign the user to an existing group, and test real access permissions.  
This helps demonstrate how users inherit permissions from groups in the **Role-Based Access Control (RBAC)** model.

## Objectives

After completing this section, you will be able to:

- Create an **IAM User**
- Assign the user to an existing group
- Log in using an IAM User
- Test S3 access (Allow and Deny)

---

## Step 1: Create IAM User & Add to Group

Go to:

- **IAM**
- **Users**
- **Create user**

At this step:

- Enter username: **dev-2**
- Select: **Provide user access to the AWS Management Console**
- Choose: **Add user to group**
- Select group: **test-policy**

![user](/images/4-Workshop/4.3-User-and-Test/1.png)

---

## Step 2: Complete User Creation

- After creation, the user **dev-2** will appear in the user list.

![user](/images/4-Workshop/4.3-User-and-Test/2.png)

---

## Step 3: Enable Console Access and Get Login Info

Go to:

- **IAM** > **Users** > **dev-2**

Do:

- Enable **Console access**
- Choose **Autogenerated password**
- Copy **Sign-in URL**, username, and password

![login](/images/4-Workshop/4.3-User-and-Test/3.png)

---

## Step 4: Login with IAM User

- Open the **Sign-in URL**
- Log in using user **dev-2**

![login](/images/4-Workshop/4.3-User-and-Test/4.png)

---

## Step 5: Access Amazon S3

- After logging in, navigate to **Amazon S3**
- The user can:
  - View bucket list
  - Open buckets

![s3](/images/4-Workshop/4.3-User-and-Test/5.png)

---

## Step 6: Test Restricted Actions (Access Denied)

Do:

- Try uploading a file to S3

Result:

- The system returns **Access Denied**

![deny](/images/4-Workshop/4.3-User-and-Test/6.png)

---

## Conclusion

- The user **dev-2** can access and read data in S3
- The user cannot upload or modify resources
